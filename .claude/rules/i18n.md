# Rules — i18n

## Structure
- **One JSON file per locale:** `src/i18n/locales/en.json`, `de.json`, `ru.json`.
- Feature translations are **top-level sections** within each file — not separate files or namespaces (e.g. `auth.*`, `home.*`, `settings.*`).
- The section name must match the feature directory name, lowercase.

## Usage
```tsx
const t = useT()                  // no argument — returns t directly
t('auth.login.title')             // full dotted path: section.key[.subkey]
t('home.welcome', { name })       // interpolation
```
`useT()` is a thin wrapper around `useTranslation()` that returns `t` — it takes no namespace argument.

## Must
- All user-facing strings go through `t()` — no hardcoded copy in JSX or styles.
- After adding or changing keys: run `npm run i18n:all` (extract + generate types).
- Add translations for **all three locales** (`en`, `de`, `ru`) — missing keys fall back to `en` but parity is required.
- Keep RTL layout in mind when adding new screens (avoid hard-coded `left`/`right` alignment).

## Must not
- Do not call `useT('namespace')` with an argument — the hook signature is `useT()`.
- Do not import translation JSON directly — use `useT()` / `t()`.
- Do not hardcode strings in `StyleSheet.create()` or component defaults.
